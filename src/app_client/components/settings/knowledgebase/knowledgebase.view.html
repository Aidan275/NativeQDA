Terminology

"File"

A file within the NativeQDA system is conistant with what you would normally conisder a file in other system. These are individual

files which you upload through the file upload page and can be in a variety of types such as pdf, docx, txt, mp3 ,jpg e.t.c. These

files are used to create datasets and are displayed on the map.

"Dataset"

A dataset is a collection of multiple files that can be created through the Datasets page. NativeQDA is built to perform analysis

on multiple files, Datasets exist to organize these files into groups which are easy to browse and edit to create an analysis.

"Analysis"

An analysis is conducted on datasets to generate visualisations and can be created in the Analysis page. An analysis is this process

of extracting relationships between word frequency, entites, concepts, themes e.t.c. After an analysis has been created from a datsets

all the available visualisations which are combatable with that dataset will automatically be avaialable int eh visualisations page for viewing

"Visualisation"

A visualisation occurs as a result of an analysis and comes in forms of any charts, graphs, diagrams or visual representation of the information

which was extracted and created during the analysis process. These visualisations can be saved for use in other research.
